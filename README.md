# PixivVault

PixivVault は、指定したPixivユーザーのイラスト・マンガ作品を一括でローカルに保存し、継続的に管理するための堅牢なバックアップ（ダウンロード）ツールです。
直感的なGUIを備え、一度ダウンロードした作品はスキップする「差分ダウンロード」や「削除検知」など、コレクター向けの運用に特化した機能を搭載しています。

## 主な機能

* **差分ダウンロード**
  ローカルのSQLiteデータベースでダウンロード履歴を管理。更新や新規追加された作品のみを高速にダウンロードします。
* **削除作品の検知**
  Pixiv上で非公開・削除されてしまった作品をローカルDBと比較して検知し、ログに警告を表示します。
* **完全ダウンロードモード**
  データベースの記録を無視し、全作品のメタデータと画像の存在チェック・再ダウンロードによる整合性修復を行います。
* **エクスポート機能**
  画像ファイル、SQLiteデータベース、全メタデータのJSONダンプを一つのZIPファイル（`PixivVault_Backup_YYYYMMDD.zip`）として書き出します。
* **安全で確実な保存処理**
  * 通信エラー時の自動リトライと指数バックオフ。
  * Pixivサーバーへの負荷を軽減するランダムスリープ。
  * ダウンロード中の破損を防ぐ一時ファイル（`.tmp`）機構と確実なトランザクション管理。
  * 任意のタイミングで処理を中断できる「一時停止」「停止」ボタン。
* **モダンなGUI**
  Fletを採用したクロスプラットフォーム対応のダークモードUI。

## 使い方

### 実行ファイルの起動
本リポジトリのReleaseページ等から `PixivVault.exe` をダウンロードし、ダブルクリックで起動します。

### 1. Cookieのインポート
PixivのAPI仕様上、作品のダウンロードにはブラウザのログインCookieが必要です。
1. ChromeやEdgeなどのブラウザでPixivにログインします。
2. 拡張機能（[Get cookies.txt LOCALLY](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc) など）を使用して、Cookieを `cookies.txt` として書き出します。
3. PixivVaultの右上にある「設定（歯車アイコン）」をクリックし、エクスポートした `cookies.txt` をインポートしてください。

### 2. ダウンロードの実行
1. **PixivユーザーID**の欄に、保存したい作者のユーザーID（PixivのURL `users/XXXXX` の数字部分）を入力します。
2. **実行モード**（差分ダウンロード / 完全ダウンロード）を選択します。
3. **実行ボタン**を押すとダウンロードが開始されます。

> **Note**: ダウンロードされた画像は、実行ファイルと同じ階層に作成される `Images/` フォルダ内に作者ごとに保存されます。

## 開発者向けセットアップ

ソースコードから実行・ビルドする場合は、以下の手順に従ってください。

### 必要要件
* Python 3.10 以上

### インストール

```bash
# リポジトリのクローン
git clone https://github.com/your-username/PixivVault.git
cd PixivVault

# 依存パッケージのインストール
pip install requests flet
```

### 実行

```bash
python main.py
```

### 実行ファイル（exe）のビルド

PyInstallerを使用して単一のexeファイルにパッケージングできます。

```bash
pip install pyinstaller
flet pack main.py --name PixivVault
```

ビルドが完了すると、`dist/PixivVault.exe` が生成されます。
